# 課題：アトミックなファイル操作マネージャ (Atomic File Transaction)

システム開発において、複数のファイルに対する変更を一貫性を持って（All or Nothingで）適用したい場面があります。例えば、設定ファイルとデータファイルを同時に更新する場合、片方だけ更新されてもう片方が失敗すると、データの不整合が発生します。

本課題では、Pythonのコンテキストマネージャを活用して、ブロック内で行った複数のファイル操作を「トランザクション」として扱い、エラーが発生した場合には全ての操作をロールバックする仕組みを構築してください。

## 要件

1. **トランザクション制御**:
   - `with FileTransaction() as tx:` のような形式で使用できること。
   - ブロックが正常に終了した場合のみ、全ての変更が実ファイルに適用（コミット）されること。
   - ブロック内で例外が発生した場合、それまでに行った変更は破棄（ロールバック）され、元のファイル状態が維持されること。

2. **サポートする操作**:
   - `tx.write(path, content)`: 指定したパスに内容を書き込む。
   - `tx.delete(path)`: 指定したファイルを削除する。
   - (オプション) `tx.move(src, dst)`: ファイルを移動する。

3. **堅牢性**:
   - 書き込み中にプロセスがクラッシュした際のリスクを最小限にするため、一時ファイル（`.tmp`など）を活用すること。
   - 既に存在するファイルを `write` する場合、元のバックアップを保持し、ロールバック時に復元できること。

4. **Pythonicな実装**:
   - `contextlib` や `pathlib` を適切に使用すること。
   - 適切な例外処理を行い、リソース（一時ファイルなど）のリークを防ぐこと。

## ヒント
- 一時的な変更を保持するために、`tempfile` モジュールや、特定の一時ディレクトリを作成して管理する手法が考えられます。
- 「コミット」フェーズでは、一時ファイルを実ファイルにアトミックに `os.replace` (または `pathlib.Path.rename`) するのが一般的です。
