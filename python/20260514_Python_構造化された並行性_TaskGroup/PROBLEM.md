  Python: asyncio.TaskGroup と ExceptionGroup による構造化された並行処理

  2. 【課題：並列データ集計システムの構築】
  複数の「不安定なデータソース」から並列にデータを取得し、集計するシステムを構築してください。

  要件：
   1. データソースのシミュレート:
       - fetch_data(source_id: int) という非同期関数を作成してください。
       - 各タスクは 0.5〜2.0秒 のランダムな時間待機します。
       - source_id == 3 の場合のみ、意図的に RuntimeError を送出させてください。
   2. タスク管理:
       - asyncio.gather ではなく、asyncio.TaskGroup を使用して、5つのタスク（ID: 1〜5）を並行実行してください。
   3. エラーハンドリング:
       - TaskGroup 内で例外が発生した際、他の「まだ動いているタスク」がどうなるかを観察してください。
       - Python 3.11 の新構文 try...except* (ExceptionGroup の一括キャッチ)
         を使用して、失敗したタスクの情報を表示してください。
   4. 結果の集約:
       - 成功したタスクの結果（例えば "Data from {source_id}" という文字列）をリストに格納し、最終的に表示してください。
